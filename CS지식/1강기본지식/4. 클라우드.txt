1. 가상화 배포방식
  1) 가상머신을 기반으로 배포하는 것을 말합니다.
    - 가상머신이란 컴퓨터의 하드웨어를 소프트웨어적으로 구현한 것을 말합니다.
  2) 중간에 있는 하이퍼바이저는 하나의 시스템 상에서 가상 컴퓨터를 여러 개
     구동할 수 있도록 해 주는 중간 계층을 의미합니다.
     그 위에 여러개의 가상머신을 구축할 수 있습니다.(OS, 앱 등)
  3) 클라우드는 이러한 가상화 기술 때문에 한대의 하드웨어로 여러명의 
     사용자들에게 독립적으로 클라우드 서비스를 할 수 있습니다.
  4) 단점 일일히 os 설치

2. 오프프레미스, 온프레미스
  1) 오프프레미스(클라우드, aws)
    - 내가 아닌 다른 회사의 공급자가 호스팅하고 인터넷을 통해 사용자에게
      제공되는 인프라, 플랫폼 또는 소프트웨어를 말합니다.
    - 자체 인프라나 하드웨어 설치 없이 앱과 리소스에 쉽고 싸게 이용이 
      가능합니다.
  2) 온프레미스
    - 네트워크 선까는 것부터 시작해 서버, 데이터베이스 설치 등을 하는 것
    - 프리쿨링을 통한 전기 절약이 가능합니다
      - 서버실 온도를 조절하기 위해 냉각수가 아닌 찬공기를 이용해 만든 방식

3. IaaS, PaaS, SaaS
  1) IaaS(Infrastructure-as-a-Service) -> EC2 -> 빈방
    - 인프라형 클라우드 서비스
    - 클라우드가 단지 인프라를 제공합니다.
    - 자바, 스프링 등 직접 설치해야 됩니다.
  2) PaaS(Platform-as-a-Service) -> 헤로쿠 -> 빌트인
    - 플랫폼형 클라우드 서비스
    - 클라우드가 플랫폼을 제공합니다.
    - 자바, 스프링 등 설치가 되어있어 단순 클릭으로 설치가 가능합니다.
    - 모니터링, CI/CD가 제공됩니다.
    - 컴포넌트로 구성되어 있어 플랫폼에 종속적임
      - 그래서 유연성, 이식성 낮음
      - 설치가 쉽고 모니터링 등이 무료로 제공되서 운영비효율이 좋음

  3) SaaS(Software-as-a-Service) -> 구글DOCS
    - 서비스형 클라우드 서비스
    - 완전한 서비스를 클라우드 서비스로부터 제공받아 사용합니다.

  4) IaaS Vs PaaS
            IaaS    PaaS
    유연성  높음    낮음
    이식성  높음    낮음
운영비효율  낮음    높음

4. 컨테이너와 도커
  1) 컨테이너 - 하나하나 옮기는 게 아니라 한꺼번에 싸서 옮기는 것
    - 애플리케이션이 한 컴퓨팅 환경에서 다른 컴퓨팅 환경으로 빠르고 안정적으로
      실행되도록 코드와 모든 종속성을 패키징하는 소프트웨어의 표준 단위입니다.
    - OS를 공유하기 때문에 빠르고, 경량화되있으며 격리성도 좋다.
    - 단점 : OS에 문제가 생기면 다른 앱에도 영향을 미칠 수 있다.

  2) 도커
    - 컨테이너 배포에 필요한 거의 모든 기능을 제공하는 플랫폼
      - IaaS와 PaaS의 장점만 가져옴
    - 도커의 과정
      [1] 도커파일 : 패키지, 환경변수 설정 등을 기록한 파일
                        |  
                        ∨ 빌드
      [2] 도커이미지 : 컨테이너 실행에 필요한 파일과 설정값, 데이터 등을
                       포함한 상태값이며 불변합니다. 
                       하나의 이미지에서 여러개의 컨테이너를 생성할 수 있음
                        |  
                        ∨ 실행
      [3] 도커컨테이너 : 컨테이너가 실행되면 도커이미지에 설정된 프로그램,    
                         데이터 등이 실제 컴퓨팅 자원과 연결됩니다.
