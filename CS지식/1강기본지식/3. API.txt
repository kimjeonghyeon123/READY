1. API(Application Programming Interface)
  1) 둘 이상의 컴퓨터 프로그램이 서로 통신하는 방법입니다.
  2) 프로토콜, 메서드, 데이터타입 등이 정의된 중계 계층을 의미합니다.
    - 프로토콜 : HTTP, HTTPS
    - 메서드 : GET, POST...
    - 데이터타입 : JSON, XML
  3) 인터페이스
    - 서로 다른 두 개의 시스템, 장치 사이에서 정보나 신호 주고받는 경우의 
      접점입니다.
    - 이를 통해 해당 컴퓨터의 내부서버가 어떻게 구현되어있는지 상관없이 
      인터페이스를 통해 통신 등이 가능
  4) 장점
    - 제공자는 서비스의 원하는 부분만 보여줄 수 있습니다.
    - 사용자는 서비스가 어떻게 구현되있는지 알 필요 없이 필요한 정보만 
      받을 수 있습니다.
    - open api 경우 앱 개발 프로세스를 단순화시키고 시간과 비용 절약가능
      - ex) 네이버 로그인 api 사용
    - 내부 프로세스가 수정되었을 때 api가 수정 안되게 할 수 있음
      - db 튜닝을 진행해도 사용자는 모른다. api를 통해 사용하기 때문이다.
    - 제공자는 데이터를 한 곳에 모을 수 있습니다.
    - private : 회사 내부, public : 모든 사람
