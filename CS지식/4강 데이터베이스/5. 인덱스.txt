1. 인덱스
  1) 데이터를 빠르게 찾을 수 있는 하나의 장치
  
  2) 균형잡힌 B-Tree 기반으로 구축되어 있어 탐색에 평균 O(logN) 시간이 걸림
  
  3) 인덱스를 설정할 때
    - 같음, 정렬, 다중 값, 유니크 한 정도 순으로 설정해야 됨
    - 쿼리문에 == 이 많은 것
    - 정렬을 많이 수행하는 것
    - 다중 값이 많은 것
    - 유니크한 값인 것
  
  4) 클러스터드 인덱스
    - 유일성과 최소성을 가지는 기본키 중 하나로 설정
    - 테이블 당 한개
    - 보통 기본키가 클러스터형 인덱스가 됨
    - 정렬이 되어 있어 탐색이 빠르다
    - 데이터가 추가될 때마다 모든 테이블을 정렬해야 하기 때문에 삽입, 삭제
      수정이 느림
  
  5) 넌크러스터드 인덱스
    - 보조 인덱스라고도 함
    - 여러개를 가질 수 있음
    - 복합키를 설정할 때 자주 함
    - 데이터 페이지에 관한 포이터가 있음
    - 정렬되어 있지 않아서 탐색은 느리나 삽입, 삭제가 빠름
    - 인덱스의 순서와 데이터의 순서가 일치하지 않음

2. B-트리
  1) 루프 노드, 리프 노드, 브랜치 노드로 구성되어 있음
  
  2) 이진 트리를 확장해 하나의 노드가 가질 수 있는
     자식 노드의 숫자가 2보다 큰 트리 구조를 가진 균형잡힌 트리
