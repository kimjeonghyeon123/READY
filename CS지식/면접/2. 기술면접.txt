1. HTTP 메서드에 대해 설명해 보세요.

데이터에 대해 서버가 수행해야 할 동작을 지정하는 방법입니다.
HTTP METHOD에는 get, post, put, delete 가 있습니다.

get은 클라이언트에서 서버로 정보를 요청하기 위해 사용하는 method입니다.
서버에 데이터를 전달하는 경우 쿼리스트링을 통해 전달합니다.?&
브라우저에도 기록이 남고 url에 정보가 노출된다는 것을 유의해야합니다.

post는 데이터를 생성하기 위해 사용하는 method 입니다.
http 메세지 body에 키 밸류 형식으로 전달합니다.

delete는 데이터를 삭제하기 위해 사용하는 method 입니다.

put과 patch는 데이터를 수정하기 위해 사용하는 method입니다.
put은 덮어쓰기, patch는 바뀐 부분만 바꾸는 것이 차이점입니다.


2. HTTP 상태 코드에 대해 설명해 보세요.

HTTP 요청이 성공적으로 완료되었는지 알려줍니다.
응답은 5가지로 나뉘어집니다.
1로 시작하는 응답은 요청을 받았고 작업을 계속한다를 의미합니다.
100은 요청을 첫 부분을 받았고 나머지를 기다리고 있음을 의미합니다.
2로 시작하는 응답은 요청을 성공적으로 받아서 처리했다라는 것을 의미합니다.
200은 요청을 제대로 처리했다라는 것을 의미합니다. 주로 get 요청입니다.
3으로 시작하는 응답은 요청을 마치기 위해 추가 동작을 취해야 한다라는 것을 
의미합니다.
4로 시작하는 응답은 클라이언트에 오류가 있다는 것을 의미합니다.
404 서버가 요청한 페이지를 찾을 수 없다라는 것을 의미합니다.
5로 시작하는 응답은 서버에 오류가 있다는 것을 의미합니다.
503  : 서버가 과부하되거나 다운되어서 사용할 수 없다라는 것을 의미합니다.


3. 프로세스와 스레드의 차이점에 대해 설명해 보세요.

프로세스는 운영체제로부터 자원을 할당받는 작업의 단위이고 스레드는 프로세스로부터 자원을 할당받아 실행하는 작업의 단위입니다.
프로세스는 다른 프로세스와 자원을 공유하지 않습니다. 다른 프로세스의 자원에 접근하기 위해서는 프로세스 간 통신 IPC 등을 사용해야 합니다. 
하나의 프로세스에서 문제가 생겨도 다른 프로세스가 영향을 받지 않습니다.
Context 스위칭 비용이 큼 오버헤드가 발생할 수 있습니다.
스레드는 다른 스레드와 자원을 공유합니다.
작업의 단위가 작아 Context 스위칭 비용이 작습니다.
하나의 스레드에서 문제가 생기면 전체 프로세스가 영향을 받습니다.

자바 스레드는 JVM이 운영체제의 역할을 하여 스레드에 대한 정보들을 관리합니다.
스레드의 개수, 스레드로 실행되는 코드의 메모리 위치 등을 관리합니다.

CPU에서 여러 프로세스를 돌아가면서 작업을 처리하는 데 이 과정을 Context Switching라 한다.


4. RDBMS와 NoSQL의 차이에 대해 설명해 보세요.

RDBMS는 정해진 스키마가 존재하고 NoSQL은 정해진 스키마가 없습니다.
RDBMS는 명확한 데이터 구조를 보장하고 중복을 배제할 수 있습니다.
테이블 간에 관계를 맺고 있어서 시스템 커질 경우 쿼리문이 복잡해지고 스키마의 변경이 어렵습니다.
NoSQL은 키, 값 쌍으로 저장되고 어떠한 형태의 데이터라도 담을 수 있고 응답속도가 빠릅니다. 데이터의 중복이 발생할 수 있습니다.


5. DB에서 인덱스를 잘 사용하면 어떤 장점이 있을까요?

인덱스를 사용하면 데이터 검색할 때 빠르다는 장점이 있습니다. 
대용량의 데이터베이스 테이블에서 필요한 데이터를 빠르게 찾기 위해 인덱스가 필요합니다. 그렇지 않을 경우 테이블 전체를 읽어야 되기 때문에 시간이 오래 걸립니다. 삽입, 삭제, 수정 등의 작업에 대해서는 성능이 저하됩니다. 인덱스는 DB 데이터 주소를 갖고 있는 것을 의미합니다.
