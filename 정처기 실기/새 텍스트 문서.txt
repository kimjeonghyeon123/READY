- SOLID
  - SRP : 하나의 객체는 하나의 책임을 가진다
  - OCP : 수정에는 닫혀있고 확장에는 열려있다.
  - LSP : 자식클래스는 최소한 부모 클래스의 기능은 수행할 수 있다.
  - ISP : 자신이 사용하지 않는 인터페이스와는 의존 관계를 맺지 말아야 한다.
  - DIP : 자신보다 추상성 높은 인터페이스와 의존 관계를 맺어야 한다

- 결합도
  - 자료 결합도 : 자료 요소로만 구성
  - 스탬프 결합도 : 배열, 자료구조 전달
  - 제어 결합도 : 논리적 흐름 제어하는 제어 신호 전달
  - 외부 결합도 : 다른 모듈의 변수 참조
  - 공통 결합도 : 전역 변수 공통으로 사용
  - 내용 결합도 : 내부 자료 직접 참조

- 응집도
  - 기능적 응집도 : 모든 기능이 단일 문제
  - 순차적 응집도 : 출력이 다음 입력으로
  - 교환적 응집도 : 같은 입출력 씀 다른 기능
  - 절차적 응집도 : 기능을 순차적으로 수행
  - 시간적 응집도 : 특정 시간에 처리되는 기능
  - 논리적 응집도 : 유사한 성격, 
  - 우연적 응집도 : 서로 관련없음

- IPC(Inter-Process Communication)

- 디자인 패턴
  - 생성 패턴
    - 추상 팩토리 : 구체적 클래스말고 추상화된 인터페이스에 의존하게 함
    - 빌더 : 객체 생성과 표현 방법 분리
    - 팩토리 메소드 : 객체 생성을 서브 클래스에서 처리하도록 캡슐화, 가상 생성자, 상위 클래스에서 인터페이스, 실제 생성 서브클래스
    - 프로토타입 : 원복 객체 복제
    - 싱글톤 : 인스턴스 하나
  - 구조 패턴
    - 어댑터 : 호환성 없는 클래스 이용할 수 있게 변환
    - 브리지 : 구현부에서 추상층 분리, 기능과 구현을 두 개로 분리
    - 컴포지트 : 복합 객체를 단일 객체와 구분 없이 다룸
    - 데코레이터 : 객체 간 결합으로 기능 확장
    - 퍼싸드 : wrapper 객체, 복잡한 서브 클래스 피해 상위 인터페이스 구성
    - 플라이웨이트 : 인스턴스 매번 생성안하고 가능한 한 공유
    - 프록시 : 접근 어려운 객체 사이 인터페이스 역할
  - 행위 패턴
    - 책임 연쇄 : 책임
    - 커맨드 : 요청을 캡슐화하여 로그에 남김, 명령어를 추상, 구체 클래스로 분리
    - 인터프리터 : 문법 표현 정의, SQL
    - 반복자 : 접근 잦은 객체에 대해 동일한 인터페이스 사용
    - 중재자 : 객체들 간 수많은 상호작용 캡슐화
    - 메멘토 : 특정 시점 객체 내부 상태 객체화, ctrl+z
    - 옵서버 : 상태 변화를 다른 객체에 전달
    - 상태 : 상태에 따라 다르게 동작
    - 전략 : 알고리즘 캡슐화
    - 템플릿 메소드 : 상위 클래스에 골격 정의, 하위 클래스에서 세부 처리
    - 방문자 : 클래스에서 처리 기능 분리, 방문 수행

  - 가용성(Availability) : 인가받은 사용자는 언제나 사용 가능
  - 부인 방지(NonRepudiation) : 
  - 인증(Authentication) : 비밀번호 등 인증 행위

- 공격
  - SQL 삽입 : 프로그램에 sql을 삽입해 서버 데이터 유출 및 변조, 관리자 인증 우회
  - XSS : 악의적인 스크립트 삽입해 정보 탈취
  - 메모리 버퍼 오버플로 : 할당된 메모리 범위 넘어선 위치에서 자료 사용
  - Ping of Death : 크기가 큰 패킷
  - Smurfing : 엄청난 양의 패킷 전송
  - SYN Flooding : 3-way handshake 중단시켜 대기 상태 지속
  - TearDrop : 패킷 순서 변경 오버헤드
  - LAND Attack : 자신에 대해 무한히 응답
  - DDos : 분산 서비스 공격, 분산된 곳에서 한 곳에 서비스 거부 공격 수행
  - 세션 하이재킹 : 세션 가로채기
  - ARP 스푸핑 : MAC 주소로
  - 스미싱 : 문자 스미싱
  - 사회 공학 : 지인 
  - 다크 데이터 : 모아두고 안쓰는 데이터
  - 타이포스쿼팅 : 도메인 비슷하게 해서
  - 스피어 피싱 : 이메일 지속적 전달
  - APT : 지능형 지속적인 위협, 악성코드 이메일 지속적 전달, 기업
  - 무작위 대입 기법(브루트 포스 공격) : 모든 값 대입
  - 큐싱 : QR코드
  - 스니핑 : 네트워크 중간에서 남의 패킷 정보 도청
  - 워터링 홀 : 자주 방문하는 웹사이트를 사전에 감염시킴
  - 좀비 PC : 감염되어 다른 컴퓨터 조종하도록 하는 컴퓨터
  - C&C서버 : 좀비 pc에 명령 내리는 서버
  - 봇넷 : 악성코드에 감염된 컴퓨터들이 네트워크로 연결된 형태
  - 웜 : 자가복제, 과부하
  - 제로 데이 공격 : 공격의 신속성
  - 키로거 공격 : 키보드 읽음
  - 랜섬웨어 : 내부 문서 잠금
  - 백도어 : 비밀 통로
  - 트로이 목마 : 정상인 척 들어와 동작할 때 활성화
